# 装饰画图片管理软件产品文档

## 1. 产品概述

### 1.1 产品名称

装饰画图片管理与多尺寸生成软件

### 1.2 产品定位

本软件面向拼多多装饰画商家，用于集中管理不同店铺的装饰画原图、图片编码、尺寸版本和打印参数。软件帮助商家快速上传图片、自动生成规范编码、批量生成不同尺寸图片，并按店铺和尺寸保存到指定文件夹，方便后续用于商品上架、订单生产、打包发货和文件迁移。

### 1.3 使用对象

- 装饰画商家
- 店铺运营人员
- 图片处理人员
- 打印生产人员

### 1.4 核心目标

- 统一管理多个店铺的图片文件
- 每张图片绑定唯一编码
- 支持自动生成图片编码
- 支持一张原图生成多个尺寸版本
- 支持自定义宽、高、DPI
- 支持修改和迁移图片保存文件夹
- 降低人工命名、找图、改尺寸的出错率

## 2. 核心业务场景

### 2.1 新图片入库

商家拿到一张新的装饰画图片后，在软件中选择所属店铺，上传图片，填写或自动生成图片编码。软件读取图片宽高、DPI 等信息，并将图片保存到该店铺对应的文件夹中。

### 2.2 一键生成多个尺寸

同一张装饰画通常有多个销售规格，例如 `20x30cm`、`30x40cm`、`40x60cm`。用户上传原图后，可以选择多个宽高和 DPI，软件自动生成对应尺寸的图片文件，并按照编码规则命名。

### 2.3 多店铺文件夹管理

用户有多个拼多多店铺，每个店铺需要对应独立图片文件夹。软件支持为每个店铺设置保存路径，并支持后期修改路径或迁移图片文件。

### 2.4 后续扩展场景

未来可扩展为读取拼多多未发货订单表，根据订单商品编码自动匹配对应图片，并生成生产清单或打包文件。

## 3. 功能范围

### 3.1 店铺管理

#### 3.1.1 新增店铺

字段包括：

#### 3.1.3 删除或停用店铺

店铺已有图片数据时，建议优先使用“停用”，避免历史图片找不到归属。

#### 3.1.4 店铺文件夹设置

每个店铺可以设置一个或多个文件夹：

- 原图文件夹
- 成品图文件夹
- 按尺寸分类文件夹


修改方式：

### 3.4 图片编码管理

#### 3.4.1 编码格式

图片编码格式为：

```text
XX-XXXX-XX-XX
```

含义：

```text
XX      店铺名称首字母大写
XXXX    四位 16 进制编号
XX      宽，单位 cm
XX      高，单位 cm
```

示例：

```text
SG-00AF-20-30
SG-00AF-30-40
SG-00AF-40-60
```

说明：

- `SG` 表示店铺名称首字母，例如“拾光”
- `00AF` 表示四位 16 进制编号
- `20-30` 表示宽 20cm，高 30cm

#### 3.4.2 自动生成编码

当用户不知道如何编码时，可以点击“一键生成编码”。

生成规则：

1. 根据当前选择的店铺生成店铺首字母。
2. 自动生成一个不重复的四位 16 进制编号。
3. 根据用户选择的宽和高拼接尺寸。
4. 形成完整编码。

示例：

```text
SG-03C8-20-30
```

#### 3.4.3 编码唯一性

软件需要保证同一店铺下，同一个基础编号不重复。

基础编号指：

```text
SG-03C8
```

同一张图不同尺寸允许共用同一个基础编号：

```text
SG-03C8-20-30
SG-03C8-30-40
SG-03C8-40-60
```

不同图片不能使用同一个基础编号。

#### 3.4.4 手动输入编码

用户也可以手动输入编码。

软件需要校验：

- 是否符合编码格式
- 店铺首字母是否和当前店铺匹配
- 编号是否为四位 16 进制字符
- 宽高是否为数字
- 是否与已有图片冲突

### 3.5 多尺寸生成

#### 3.5.1 尺寸选择

用户可以为一张图片选择一个或多个尺寸。

常用尺寸示例：

- 20 x 30 cm
- 30 x 40 cm
- 40 x 50 cm
- 40 x 60 cm
- 50 x 70 cm
- 60 x 80 cm

#### 3.5.2 自定义尺寸

用户可以手动输入宽和高。

字段包括：

- 宽度，单位 cm
- 高度，单位 cm
- DPI

#### 3.5.3 DPI 选择

用户可以选择或输入 DPI。

常用 DPI：

- 72 DPI，适合预览图
- 150 DPI，适合普通装饰画打印
- 200 DPI，适合较高清打印
- 300 DPI，适合高清打印

#### 3.5.4 像素计算规则

生成图片时，根据宽、高、DPI 计算目标像素。

计算公式：

```text
目标像素宽 = 宽 cm / 2.54 x DPI
目标像素高 = 高 cm / 2.54 x DPI
```

示例：

```text
20 x 30 cm，150 DPI
目标像素约为 1181 x 1772 px
```

#### 3.5.5 生成方式

软件支持：

- 保持比例缩放
- 居中裁剪
- 留白适配
- 拉伸适配，默认不推荐

默认建议：

```text
保持比例 + 居中裁剪
```

#### 3.5.6 清晰度提醒

当原图像素不足以生成目标尺寸时，软件需要提醒用户。

提醒示例：

```text
当前原图为 1181 x 1772 px，适合生成 20 x 30 cm / 150 DPI。
如果生成 40 x 60 cm / 150 DPI，清晰度可能不足。
```

### 3.6 文件保存规则

#### 3.6.1 文件命名

生成后的图片文件名建议使用完整编码。

示例：


#### 3.6.2 文件夹结构

```text

├─ 原图
├─ 20-30
├─ 30-40
```

```text

│  ├─ 原图.jpg
│  ├─ SG-03C8-20-30.jpg
│  ├─ SG-03C8-30-40.jpg
│  └─ SG-03C8-40-60.jpg
```

建议优先使用结构一，因为后续生产和查找尺寸更方便。

## 5. 本地记录文件设计

### 5.1 存储原则

本软件第一版不使用数据库，所有信息通过本地文件保存。

推荐使用：

- `shops.json` 保存店铺和文件夹设置
- `image_index.csv` 保存图片索引，方便 Excel 打开查看
- `settings.json` 保存软件设置
- `logs` 文件夹保存操作记录
- 图片文件本身按店铺和尺寸保存在指定文件夹

这样做的好处：

- 不需要安装数据库
- 文件夹可以直接迁移
- 图片记录可以用 Excel 打开
- 出问题时容易人工检查和修正
- 后续仍然可以升级为数据库版本

### 5.2 推荐文件结构

软件可以在一个总配置文件夹中保存记录文件。

示例：

```text
装饰画图片管理
├─ config
│  ├─ shops.json
│  ├─ settings.json
│  └─ image_index.csv
├─ logs
│  ├─ 2026-06-20.log
│  └─ 2026-06-21.log
└─ backups
   ├─ image_index_20260620.csv
   └─ shops_20260620.json
```

图片文件可以继续保存在用户指定的店铺文件夹中，不强制放在软件目录里。

### 5.3 店铺配置文件

文件名：

```text
shops.json
```

保存内容：

| 字段 | 说明 |
| --- | --- |
| name | 店铺名称 |
| short_name | 店铺简称 |
| prefix | 店铺首字母大写 |
| original_folder | 原图文件夹 |
| output_folder | 成品图文件夹 |
| enabled | 是否启用 |
| remark | 备注 |

示例：

```json
[
  {
    "name": "拾光装饰画",
    "short_name": "拾光",
    "prefix": "SG",
    "original_folder": "D:\\装饰画链接\\挂轴画\\拾光 原图",
    "output_folder": "D:\\装饰画链接\\挂轴画\\拾光 成品图",
    "enabled": true,
    "remark": ""
  }
]
```

### 5.4 图片索引文件

文件名：

字段建议：

| 字段 | 说明 |
| --- | --- |
| shop_name | 店铺名称 |
| shop_prefix | 店铺首字母 |
| base_code | 基础编码，例如 SG-03C8 |
| full_code | 完整编码，例如 SG-03C8-20-30 |
| original_name | 原始文件名 |
| original_path | 原图路径 |
| output_path | 成品图路径 |
| width_cm | 宽 cm |
| height_cm | 高 cm |
| dpi | DPI |
| width_px | 原图像素宽 |
| height_px | 原图像素高 |
| output_width_px | 生成像素宽 |
| output_height_px | 生成像素高 |
| file_format | 图片格式 |
| created_at | 上传或生成时间 |
| remark | 备注 |

## 6. 编码生成逻辑

### 6.1 店铺首字母

店铺首字母由用户设置，建议使用两位大写英文字母。

示例：

```text
拾光 -> SG
美家 -> MJ
画语 -> HY
```

### 6.2 四位 16 进制编号

编号范围：

```text
0000 - FFFF
```

最多支持每个店铺 65536 个基础图片编号。

### 6.3 编号递增方式

建议从 `0001` 开始递增。

示例：

```text
SG-0001
SG-0002
SG-0003
SG-000A
SG-000F
SG-0010
```

### 6.4 完整编码生成

完整编码格式：

```text
店铺首字母-四位编号-宽-高
```

示例：

```text
SG-0010-20-30
```

## 7. 关键流程

### 7.1 单张图片上传流程

1. 用户打开图片上传页。
2. 选择店铺。
3. 选择图片文件。
4. 软件读取图片信息。
5. 用户输入编码或点击一键生成编码。
6. 用户选择一个或多个尺寸。
7. 用户选择 DPI。
8. 软件生成预览和清晰度提醒。
9. 用户确认保存。
10. 软件保存原图并生成尺寸图。
11. 软件把图片信息写入本地索引文件。

### 7.2 批量上传流程

1. 用户选择店铺。
2. 用户批量选择图片。
3. 软件读取所有图片信息。
4. 用户选择统一尺寸和 DPI。
5. 软件为每张图片自动生成基础编码。
6. 软件批量生成成品图。
7. 软件显示成功和失败结果。

### 7.3 文件夹迁移流程

1. 用户进入店铺管理页。
2. 选择需要迁移的店铺。
3. 选择新文件夹。
4. 软件检查文件夹和磁盘空间。
5. 用户确认迁移。
6. 软件移动图片文件。
7. 软件更新本地索引文件中的路径。
8. 软件生成迁移记录。

## 8. 异常处理

### 8.1 编码重复

当编码重复时，软件提示：

```text
该编码已存在，请更换编码或自动生成新编码。
```

### 8.2 图片尺寸不足

当原图不适合目标尺寸时，软件提示清晰度风险，但允许用户继续生成。

### 8.3 文件夹不存在

当保存文件夹不存在时，软件提示是否自动创建。

### 8.4 文件保存失败

保存失败时记录失败原因，并允许用户重新生成。

### 8.5 文件重名

重名处理方式：

- 覆盖原文件
- 自动改名
- 跳过

默认建议：

```text
提示用户选择，不自动覆盖。
```

- 店铺文件夹设置
- 单张图片上传
- 自动生成编码
- 手动输入编码
- 多尺寸生成
- DPI 设置
- 图片库查询
- 打开图片所在文件夹

暂不放入第一版的功能：

- 拼多多订单表导入
- 自动匹配订单
- 云端同步
- 多人账号权限
- 复杂图片编辑

## 12. 验收标准

### 12.1 店铺管理验收

- 可以新增、修改、停用店铺
- 每个店铺可以设置独立文件夹
- 修改文件夹后，新上传图片保存到新位置

### 12.2 图片上传验收

- 可以上传 JPG、PNG、WEBP 图片
- 上传后能读取像素宽高和 DPI
- 图片能保存到对应店铺文件夹

### 12.3 编码验收

- 可以自动生成符合规则的编码
- 编码不能重复
- 手动编码输入错误时有提示

### 12.4 多尺寸生成验收

- 一张图片可以生成多个尺寸
- 生成图片文件名包含完整编码
- 生成图片的像素符合宽、高、DPI 计算结果
- 原图清晰度不足时有提示

### 12.5 文件夹迁移验收

- 可以修改店铺文件夹
- 可以选择是否迁移历史图片
- 迁移后图片库路径正确

## 13. 示例

### 13.1 店铺示例

```text
店铺名称：拾光装饰画
店铺首字母：SG
原图文件夹：D:\装饰画链接\挂轴画\拾光 原图
成品图文件夹：D:\装饰画链接\挂轴画\拾光 成品图
```

### 13.2 图片示例

原图：

```text
链接1-慢慢来一切都会变好.jpg
```

读取信息：

```text
像素：1181 x 1772 px
DPI：150
适合尺寸：20 x 30 cm
```

生成编码：

```text
SG-0001-20-30
```

生成文件：

```text
D:\装饰画链接\挂轴画\拾光 成品图\20-30\SG-0001-20-30.jpg
```

### 13.3 多尺寸示例

同一张图生成：

```text
SG-0001-20-30.jpg
SG-0001-30-45.jpg
SG-0001-40-60.jpg
```

其中 `SG-0001` 表示同一张图，后面的 `20-30`、`30-45`、`40-60` 表示不同尺寸。
